# Requirements (`REQ-NNNN`)

One file per requirement. Filename format: `REQ-NNNN-short-slug.md`.

A requirement captures **what** the user needs and **how we'll know it's done**.
It does NOT capture design, technology choices, or implementation details —
those belong in the PLAN and ADRs that follow.

## Status lifecycle

`proposed` → `accepted` → `in-progress` (once PLAN accepted) → `verified` → optionally `superseded`

A REQ cannot move to `verified` until every acceptance criterion is checked.

## Template

Copy `REQ-template.md`, rename to the next available `REQ-NNNN-slug.md`, and fill it in.
